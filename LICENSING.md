## 🔓 AGPL-3.0 License

**Best for:** Developers, researchers, open-source projects

### What You Can Do:
- ✅ Use OmniID in any project
- ✅ Modify the source code
- ✅ Study how it works
- ✅ Contribute improvements back
- ✅ Distribute modified versions

### What You Must Do:
- 📢 **Publish your modifications** under AGPL-3.0 (if distributed)
- 📢 **Offer source code access** to users (network copyleft)
- 📄 **Include the license** with any distribution
- 📝 **Document modifications** clearly with dates

### Key Requirement: Network Copyleft

If you run a modified version of OmniID on a server that users can access over the network (SaaS, API, web service), you **must provide access to the modified source code** to those users.

**Example Scenarios:**
- 🚫 Running OmniID as a web service? → Must provide source code to users
- ✅ Using OmniID internally without network access? → OK
- ✅ Modifying for internal use only? → OK

## 🤔 How to Choose

### Choose **AGPL-3.0** if:

- Your project is open-source
- You plan to publish modifications
- You want community contributions
- You prefer free licensing
- Your users expect source code access

**Examples:**
- Academic research projects
- Open-source identity systems
- Community-driven initiatives
- Government-backed transparency projects

### Choose **Commercial License** if:

- You're deploying in a closed environment
- You need to keep modifications private
- You require technical support
- You're integrating into enterprise systems
- You want legal clarity and guarantees

**Examples:**
- Government identity systems (national ID)
- Enterprise SaaS platforms
- Private banking infrastructure
- Corporate authentication systems
- Proprietary identity management

## ⚖️ Legal FAQs

### Q: Can I use OmniID under AGPL in my government project?

**A:** Yes, but you must publish any modifications under AGPL-3.0. If this conflicts with your policy, consider a Commercial License. Contact: encob.business@gmail.com

### Q: Do I need a Commercial License for internal use?

**A:** Only if you modify OmniID and run it as a network service (SaaS). Internal-only use under AGPL is acceptable.

### Q: Can I contribute to OmniID while using a Commercial License?

**A:** Yes! Contributions can be made under either license. If you contribute to the main repository, use AGPL-3.0 for that code.

### Q: What happens if I violate the AGPL license terms?

**A:** AGPL violations can result in:
- Loss of license rights
- Potential legal action
- Requirement to open-source your code
- Associated damages

### Q: Can I relicense OmniID under a different license?

**A:** No, unless you have explicit written permission from the OmniID copyright holders. This is a dual-license model, not a change in primary licensing.

### Q: Is there a time limit on the Commercial License?

**A:** Licenses are annual with automatic renewal. You can terminate with 30 days' notice. Existing deployments remain usable after termination.
